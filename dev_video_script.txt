FLOORBALL SHOT VISUALIZATION - DEV VIDEO SCRIPT (MAX 2 MIN)

=== INTRO (10 Sek) ===

- Floorball Shot Visualization Dashboard
- Web-App für Schussanalyse und Visualisierung
- Import, Speicherung, interaktive Darstellung


=== WAS MACHT DIE APP? (20 Sek) ===

1. CSV-Import Tab:
   - CSV-Dateien mit Schussdaten hochladen
   - Automatische Validierung
   - Spielname und Datum eingeben
   - Speicherung in SQLite-Datenbank

2. Dashboard Tab:
   - Spiel auswählen
   - Shot Map Visualisierung auf Floorball-Feld
   - Heatmap zeigt Erfolgsquote
   - Tooltips mit Details


=== WIE FUNKTIONIERT ES? (40 Sek) ===

Technologie:
- Frontend: JavaScript, D3.js für Visualisierung
- Datenbank: SQLite im Browser (sql.js)
- Backend: Bun Server
- Hexagonal Binning für Schussgruppierung

Workflow:
1. CSV wird geparst und validiert
2. Daten in SQLite gespeichert (games + shots Tabellen)
3. Koordinatenumrechnung: Polar zu Kartesisch
4. D3.js rendert Hexagone:
   - Farbe = Success Rate (rot/grün)
   - Größe = Anzahl Schüsse
5. Persistierung auf Server

Datenbank Schema:
- games Tabelle: Spiel-Metadaten
- shots Tabelle: 31 Felder pro Schuss
- Foreign Key Beziehung


=== TECHNISCHE DETAILS D3.js (30 Sek) ===

Wie D3 im Hintergrund läuft:
- D3.js liest Schussdaten aus SQLite
- Koordinatenumrechnung: CSV hat Polar (Distanz/Winkel), D3 braucht Kartesisch (x/y)
- d3-hexbin Plugin gruppiert nahe Schüsse in Hexagone
- Farbskala berechnet Success Rate pro Hexagon
- SVG-Rendering mit Data-Binding
- Event-Listener für Tooltips und Interaktionen

Herausforderungen bei der Erstellung:
1. Koordinatensystem-Mapping:
   - Floorball-Feld 40x20m auf SVG skalieren
   - Polar zu Kartesisch Konversion korrekt implementieren
   - Ursprung und Winkel richtig ausrichten

2. Hexagonal Binning:
   - Optimale Hexagon-Größe finden
   - Wenige Schüsse vs. zu große Hexagone balancieren
   - Performance bei vielen Datenpunkten

3. Client-Side SQLite:
   - sql.js im Browser laden (1.5MB Library)
   - Datenbank-Persistierung zwischen Sessions
   - Memory-Management für große Datasets

4. Datenvalidierung:
   - 31 CSV-Spalten korrekt mappen
   - Duplikaterkennung implementieren
   - Fehlende oder ungültige Werte behandeln
